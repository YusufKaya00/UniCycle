Bu dosyada, UniCycle projenizin gereksinimlerine ve [Example] Requirement_Specification.pdf dosyasındaki görsel stile (siyah-beyaz, gölgesiz vb.) uygun olan PlantUML kodları bulunmaktadır.

Bu kodları https://plantuml.com/plantuml/ adresine yapıştırarak diyagramları oluşturabilirsiniz.

---

### Figure 1. Simplified system model diagram

Aşağıdaki diyagram, docx'teki "Client -> Next.js Application -> Firebase / APIs" akışını ve dokümanda belirtilen 5 adımı PDF'teki numaralandırılmış döngü stiliyle birleştirerek sunar:

```plantuml
@startuml
skinparam monochrome true
skinparam shadowing false
left to right direction

rectangle "Client\n(User)" as client
rectangle "Next.js\nApplication" as nextjs
rectangle "Firebase / APIs" as firebase

client -> nextjs : 1. Login\n2. Access
nextjs -> client : 5. Real-time Updates
nextjs -> firebase : 3. Retrieve/Add Data
firebase -> nextjs : 4. Update Data/Storage

@enduml
```

*(Eğer PDF'teki gibi sadece yuvarlak numaralar isteniyorsa, şu alternatif kullanılabilir:)*
```plantuml
@startuml
skinparam monochrome true
skinparam shadowing false

node "User" as user
circle "1." as n1
circle "2." as n2
circle "3." as n3
circle "4." as n4

user -right-> n1
n1 -right-> n2
n2 -right-> n3
n3 -right-> n4
n4 -up-> user : 5.
@enduml
```

### Figure 4. User registration/authorization activity diagram

"@edu.rtu.lv" mail uzantısını içeren Google girişi akışını gösteren aktivite diyagramı:

```plantuml
@startuml
skinparam monochrome true
skinparam shadowing false

title User registration/authorization activity diagram

start
:Enter system;
:Provide Authorization details
(Google Sign-In);
if (Domain is @edu.rtu.lv?) then (yes)
  :Validate user details;
  if (Valid account?) then (yes)
    :Authorize User;
    :Access System Functions;
  else (no)
    :Register User in Database;
    :Authorize User;
    :Access System Functions;
  endif
else (no)
  :Display Error Message;
  :Deny Access;
endif
stop
@enduml
```

### Figure 5. Function activity diagram (e.g., Adding a Listing)

Satıcının Google Maps lokasyonu gibi bilgileri de ekleyerek ilan oluşturduğu ana fonksiyon akışı:

```plantuml
@startuml
skinparam monochrome true
skinparam shadowing false

title Function activity diagram (Adding a Listing)

start
:Seller enters listing info;
:Input data:
- Title, Description, Category, Price
- Images
- Location (Google Maps);
if (Is data valid?) then (yes)
  :Save listing to Firestore;
  :Upload images to Storage;
  :Display success message;
  :Show updated listings;
else (no)
  :Display error message;
endif
stop
@enduml
```
